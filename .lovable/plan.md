## Problem

Clicking "Too crowded" updates the URL to `/app/event/$eventId/break/$roomId` but the screen still shows the event page. The break-room route file exists and matches, but its parent `src/routes/_app.app.event.$eventId.tsx` is a page component with no `<Outlet />`, so the child route has nowhere to render.

## Fix

Split the current `$eventId` route into a pathless layout + an index page:

1. **Rename** `src/routes/_app.app.event.$eventId.tsx` → `src/routes/_app.app.event.$eventId.index.tsx`
   - Update its `createFileRoute` path from `/_app/app/event/$eventId` to `/_app/app/event/$eventId/`.
   - No other changes — this keeps the existing event map / chat as the page shown at `/app/event/:eventId`.

2. **Create** new `src/routes/_app.app.event.$eventId.tsx` as a minimal layout:
   ```tsx
   import { createFileRoute, Outlet } from "@tanstack/react-router";
   export const Route = createFileRoute("/_app/app/event/$eventId")({
     component: () => <Outlet />,
   });
   ```

With this, `/app/event/e1` still renders the event page (via the new index route), and `/app/event/e1/break/room-04` renders the break room inside the layout's `<Outlet />`. The existing `_app.app.event.$eventId.break.tsx` smart-pick redirect and `_app.app.event.$eventId.break.$roomId.tsx` break-room screen need no changes.

`src/routeTree.gen.ts` is auto-regenerated by the Vite plugin.

## Out of scope

No changes to `BreakRoomPicker`, break-room UI, chat, or data.
